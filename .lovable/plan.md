## Goal
Replace generic "Next Likely Play" (pitch/play archetypes) with an **AI-driven Live Prop Prediction Feed**: ranked sportsbook props the AI thinks are most likely to hit on the **very next play**, generated by Gemini watching live PBP + game state + active prop lines.

## Architecture

```text
live_game_state ─┐
live PBP feed  ──┼──> edge fn: live-next-play-predictor ──> live_next_play_predictions (new table)
live_prop_quotes ┘                  (Gemini)                              │
                                                                          ▼
                                                              TerminalView (realtime)
```

## Changes

### 1. New table `live_next_play_predictions`
Migration with grants + RLS + realtime publication.
Columns: `id`, `event_id`, `player_name`, `prop_type`, `prop_label`, `line`, `side` (Over/Under), `book`, `american_price`, `prob_next_play` (0..1), `edge_pct`, `rationale` (1-line), `state_context` (json: inning/quarter/possession/count), `model`, `created_at`, `expires_at`.
Index: `(event_id, created_at desc)`.

### 2. New edge function `live-next-play-predictor`
- Triggered every 15s per active viewer (client invoke) and also by realtime PBP changes.
- Input: `event_id`.
- Pulls: latest `live_game_state`, last 5 PBP entries via `useLivePBP` source, top ~40 active props from `live_prop_quotes` (latest snapshot per player/prop/book/line, filtered to players currently in the active situation — at-bat batter+pitcher, on-court 5, on-field offense/defense), and recent `market_signals`.
- Calls Gemini (`google/gemini-3-flash-preview`) via Lovable AI Gateway with structured `Output.object` schema returning `predictions: { player_name, prop_type, side, line, book, prob_next_play, rationale }[]` (max 5).
- Prompt: "You are watching this live game. Given the current situation [game state + last plays] and the available sportsbook props, predict which props are most likely to resolve on the very next discrete play (at-bat/possession/snap). Rank by likelihood. Return at most 5."
- Edge math: combine `prob_next_play` with de-vigged fair % to compute `edge_pct`.
- Upserts into `live_next_play_predictions` with `expires_at = now() + 60s`. Old rows pruned (`delete where expires_at < now()`).
- Skip-and-return early when no active props or PBP unavailable.

### 3. New hook `useNextPlayPredictions(eventId)`
- Initial fetch of unexpired predictions ordered by `prob_next_play desc`.
- Realtime subscribe to inserts on `live_next_play_predictions filter=event_id`.
- Calls the edge function on mount + every 15s while tab is visible.
- Returns `{ predictions, lastRun, loading, error }`.

### 4. Redesigned `NextPlayPanel` (in `panels.tsx`)
- Header: "Next Play Props" + small "AI · {N}s ago" chip.
- Each row (full-width prop card, not a thin bar):
  - Player name + jersey + position
  - **Bold prop line**: e.g. `Total Bases Over 1.5`
  - Book badge + American price
  - Right side: **Likelihood %** (big), **Edge %** (green/red)
  - Sub-line: one-line AI rationale (italic, dim)
  - Click → adds to bet slip / parlay builder (future, not in this PR)
- Empty state: "AI is watching… first predictions in ~15s."

### 5. Keep `InvolvementPanel` (user wants it)
- No change to its data source; remains complementary.

### 6. `TerminalView` wiring
- Add `useNextPlayPredictions(state.game_id)` and pass `predictions` down to `TerminalPanels` (which forwards to `NextPlayPanel`).
- Drop the existing `coach_game_tendencies`-derived `nextPlay` data path from `mockFeed.ts` for this panel (keep for any other consumers — but currently only this panel uses it; safe to remove from `buildMockTerminal`).

### 7. Bug fix (runtime error noticed)
- Current React "Should have a queue" error: fix the realtime channel cleanup in `usePropQuotes` (recent edit added a channel ref but cleanup may double-unsubscribe under StrictMode). Wrap the channel in a ref-based guard so cleanup only calls `removeChannel` once.

## Technical details

- **Model**: `google/gemini-3-flash-preview` via `ai-gateway` helper in `_shared/ai-gateway.ts` (create if missing).
- **Structured output**: use `Output.object` with a small Zod schema (5 props, no enums beyond `side: "Over" | "Under"`).
- **Auth**: edge function uses `verify_jwt = false` default; reads `LOVABLE_API_KEY` from env.
- **Cost control**: Cache last prediction batch per `event_id` for 12s in-function; skip Gemini call if cache fresh.
- **Realtime**: `ALTER PUBLICATION supabase_realtime ADD TABLE public.live_next_play_predictions`.
